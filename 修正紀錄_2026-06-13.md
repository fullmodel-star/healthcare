## 第一輪：AI 多代理健檢修正（v1.5.0）

派出 4 個小幫手分頭檢查（健康算式、程式 bug、UX、PWA/資料安全）＋ 1 個整合報告。

### 🔴 高嚴重度（已修）
1. **熱量平衡算錯**：新增 `maintenanceTDEE()`（身體真實維持消耗，不含減重/增重折扣）。總覽今日結餘、每週累積淨熱量、AI 週統計改用真實維持消耗判斷盈虧，修正「折扣被算兩次」造成的會瘦/會胖系統性誤判。每日「目標熱量」仍用 `calcTDEE()` 顯示。
2. **拍照辨識失敗偷塞假資料**：失敗（無金鑰/額度/網路）時改顯示明確失敗訊息並引導改用搜尋/手動，不再自動填入固定假餐點與綠色「加入記錄」鈕。
3. **減重目標無安全下限**：減重不再無條件扣 500 kcal，加下限（男 1500／女 1200），達下限會標註。
4. **誤刪救不回**：飲食/運動/飲水刪除改為先確認；刪除鈕點擊範圍放大。
5. **日期 UTC 時區陷阱**：新增 `ymd()`，全面改用本機時區，修正台灣清晨把「今天」算成「昨天」或漏記的問題。

### 🟡 其他（已修）
- API Key 改放 `x-goog-api-key` 請求標頭，不再放在網址（避免外洩）。
- 修正 `fmtDate()` 忽略傳入日期、永遠回今天的潛在 bug。
- BMI／掃描結果加防呆，避免 Infinity／NaN。
- 「單日 ≈ ±X kg」改為「脂肪約 ±X kg（長期估算）」。
- SW 快取 v8 → v9；修正 SW 內殘留的「Anthropic」舊註解（實際用 Google Gemini）。

## 第二輪：多人物 UX 試用改版（v1.6.0）

派出 5 種不同性格的假使用者試用：👵長輩、⏱️忙碌上班族、💪健身狂、😰焦慮新手、🕵️隱私控。
留存判斷：5 人中 3 人會留，2 人（長輩、上班族）在放棄邊緣，關鍵在「開場門檻」。

### 降低開場門檻（已做）
- **飲食頁預設改「搜尋」**：不需金鑰即可記錄，拍照不再是強制第一關。
- **餐別依時間自動選**：中午打開自動跳午餐。
- **「目前體重」可直接輸入**：改了就自動記一筆並同步，移除嚇人的「請按我」大按鈕。

### 各族群改善（已做）
- **新手**：首次顯示「示範資料」橫幅＋「清掉示範，從頭開始」一鍵鈕；喝水補「已記錄 +Xml」提示。
- **長輩**：底部導覽放大（圖示 22→26px、字 10→12px、列高 64→70px）；BMR/BMI/kcal 加中文白話。
- **健身族**：開放自訂「蛋白質目標（g/kg 體重）」，飲食頁依此計算，不再寫死 ×1.8。
- **隱私控**：新增「離線模式」總開關；新增「資料會送到哪裡」說明；清除資料時詢問是否一併清 API Key。
